ProjectFlow v8

Sistema de gestão de projetos avançado com Kanban inteligente, IA generativa, diagramas interativos, Wiki colaborativa e integração nativa com Supabase.

Vanilla JS Supabase + PostgreSQL BPMN Claude AI RLS Multi-tenant Single-page App

Visão Geral

O ProjectFlow v8 é uma aplicação web single-page (SPA) para equipes de desenvolvimento e agências digitais. Integra em uma única interface todas as etapas do ciclo de vida de um projeto: da concepção ao documento executivo entregue ao cliente.

O diferencial é a fusão entre Kanban visual e o modelo de processos BPMN, que garante rastreabilidade completa de cada transição de estado. O módulo de IA Generativa (Claude API) produz documentação executiva analisando automaticamente o estado atual das tarefas.

Decisões de Design

  • Zero frameworks front-end: JavaScript vanilla com módulos em arquivos separados. Elimina dependências de build e reduz complexidade.
  • Supabase como BaaS: Backend completo (auth, banco, RLS, Edge Functions) sem servidor dedicado.
  • Row Level Security: Segurança imposta no banco de dados via RLS. Mesmo com vulnerabilidade no front-end, dados de outros usuários permanecem inacessíveis.
  • Optimistic UI: O SyncManager aplica mudanças locais antes da confirmação do servidor — resposta instantânea na interface.
  • BPMN como estado canônico: bpmn_status é o campo de controle oficial. kanban_col é derivado via trigger, garantindo consistência até em inserções SQL diretas.

Instalação & Setup

  1. Criar projeto Supabase
    Acesse supabase.com, crie um projeto gratuito e anote a Project URL e a anon/public key em Settings → API.
  2. Executar o schema SQL
    No Supabase, abra SQL Editor e execute sql/producao.sql. O script é idempotente e instala todas as 26 tabelas, ENUMs, triggers, RLS e dados seed.
  3. Deploy da Edge Function (para IA)
    supabase functions deploy claude-proxy --no-verify-jwt
    Configure: supabase secrets set ANTHROPIC_API_KEY=sk-ant-...
  4. Abrir a aplicação
    Sirva via servidor HTTP estático ou abra index.html diretamente. Na tela de login, clique em ⚙ Configurar Supabase e insira URL e Key.
  5. Seed de teste (opcional)
    Após fazer login, execute sql/v8_test_seed.sql no SQL Editor para popular o banco com 3 projetos, 12 tarefas, diagrama e docs. Recarregue a página.

📋 Módulo Kanban

O Kanban é o módulo central. Cada projeto tem um board dedicado com colunas dinâmicas, WIP limits e sincronização em tempo real. Um board padrão é criado automaticamente para cada novo projeto via trigger handle_new_project.

Roteamento de Cards — Prioridade da Lógica

  1. UUID de coluna (column_id): se presente e válido, o card aparece exclusivamente nessa coluna.
  2. BPMN mapping: fallback para cards sem UUID (modo demo ou dados legados).
  3. Campo col: fallback final para compatibilidade retroativa.

SyncManager — Atualizações Otimistas

Cada operação de escrita passa pelo SyncManager que: (1) aplica a mudança localmente e re-renderiza, (2) persiste no Supabase em paralelo, (3) faz rollback automático se o servidor retornar erro. O usuário vê a resposta instantânea sem aguardar a rede.

⚠ Trigger de Consistência

O trigger map_bpmn_to_col (BEFORE INSERT OR UPDATE OF bpmn_status) mantém o campo kanban_col sempre sincronizado com bpmn_status, inclusive em inserções SQL diretas. O trigger task_audit (AFTER UPDATE) registra em task_history qualquer mudança de status ou título.

📊 Dashboard

Visão analítica em tempo real do projeto selecionado, calculada a partir dos cards carregados em PFBoard.cards.

  • Contadores: Total, Concluídas, Em Execução, Em Revisão, Bloqueadas, Atrasadas
  • Progresso Geral: barra animada — concluídas ÷ total × 100%
  • Por Prioridade: gráfico de barras (critical → high → medium → low)
  • Por Coluna: distribuição das tarefas em cada estágio
  • Tarefas Ativas: lista de tarefas em Execução e Revisão, clicáveis

✦ Doc IA (Claude)

O módulo Doc IA usa a Claude API para gerar documentação executiva completa a partir da análise automática das tarefas do projeto.

Pipeline de 4 Chains

  1. Chain A — Contexto & Riscos: Analisa tarefas, identifica escopo e lacunas de documentação.
  2. Chain B — Stack & Arquitetura: Identifica stack tecnológica, decisões de arquitetura e artefatos.
  3. Chain C — ROI & Negócio: Estima ROI, payback e alinhamento com objetivos de negócio.
  4. Chain D — Plano de Ação: Gera resumo executivo com próximos passos priorizados e blockers.
Pré-requisito

Requer a Edge Function claude-proxy deployada no Supabase com o secret ANTHROPIC_API_KEY configurado. Sem ela, o módulo exibe estado de "Motor não configurado" mas não interfere nos demais módulos.

🗂 Banco de Documentação

Agrega em visão centralizada todos os campos de documentação preenchidos nas tarefas de todos os projetos — um repositório pesquisável de decisões, artefatos e riscos.

Filtros disponíveis: projeto, seção (Identificação / Execução / Descrição / Documentação / Anexos), status BPMN e busca textual livre.

O botão Exportar PDF gera um documento HTML completo (nova aba, pronto para impressão) com capa, métricas, barra de progresso, diagrama de arquitetura embutido e todas as tarefas documentadas.

🗄 Projetos Legados

Projetos com is_legacy = true aparecem no módulo Legados — projetos importados retroativamente (engenharia reversa) que mantêm histórico mas não possuem workflow ativo.

🔒 RLS & Segurança

Todas as 26 tabelas têm RLS habilitado. O modelo de acesso é ownership + membership:

  • Projetos: visível para owner_id e membros em project_members
  • Tarefas/Kanban: visível para membros do projeto pai
  • Perfis: leitura por todos os usuários autenticados (necessário para atribuição)
  • task_history: somente leitura via RLS; escrita exclusiva via trigger SECURITY DEFINER

Funções helper is_project_member() e can_write_project() são STABLE SECURITY DEFINER — executam com permissões elevadas mas retornam apenas booleano, evitando recálculo em joins complexos.

🔌 APIs & Integrações

Supabase JS Client

SDK oficial @supabase/supabase-js@2 via CDN. Credenciais em localStorage (pf_sb_url / pf_sb_key).

// Exemplo: buscar projetos do usuário
const { data, error } = await PF.supabase
  .from('projects')
  .select('id, name, color, status')
  .order('created_at', { ascending: true });
if (error) { showToast('Erro: ' + error.message, true); return; }

Padrão SyncManager

await SyncManager.execute(
  cardId,        // ID único — previne concorrência
  () => { /* mutação local otimista */ },
  async () => { return PF.supabase.from('tasks').update({...}); },
  () => { /* rollback se servidor retornar erro */ },
  'Alteração salva com êxito'  // toast de sucesso
);

🤖 Claude Proxy (Edge Function)

Edge Function Deno que serve como proxy seguro para a Anthropic API, resolvendo CORS e ocultando a chave de API.

// Localização: supabase/functions/claude-proxy/index.ts
// Deploy: supabase functions deploy claude-proxy --no-verify-jwt
// Secret: supabase secrets set ANTHROPIC_API_KEY=sk-ant-...

// Chamada do front-end (pf-producao.js)
const res = await fetch(supabaseUrl + '/functions/v1/claude-proxy', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + supabaseAnonKey,
    'apikey': supabaseAnonKey
  },
  body: JSON.stringify({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 1500,
    system: 'Prompt do sistema...',
    messages: [{ role: 'user', content: '...' }]
  })
});
✅ Segurança

A ANTHROPIC_API_KEY nunca é exposta ao browser. O proxy valida o JWT do Supabase antes de encaminhar à Anthropic.
